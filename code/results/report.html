1. Overview

This project implements a causal Transformer decoder language model from scratch for next-token prediction on the Penn Treebank (PTB) dataset. Key components are manually implemented using basic PyTorch modules:

  • Token embedding layer
  • Rotary Positional Embedding (RoPE) — relative position encoding via rotation matrices
  • Multi-head self-attention with causal masking
  • SwiGLU feed-forward network
  • Pre-norm LayerNorm + residual connections
  • Cosine annealing learning rate schedule with gradient clipping

The model achieves a best validation perplexity of 111.94 with 3.6M parameters, trained for 8 optimal epochs on an NVIDIA RTX 4090 GPU.

3. Rotary Positional Embedding (RoPE)

RoPE encodes relative positional information by rotating the query and key vectors in the complex plane. For a head dimension d, we precompute frequency vectors:

Frequency precomputation:

freqs[i] = 1 / (10000^(2i/d)), i = 0, 1, ..., d/2 - 1

Rotation for position t and dimension pair i:

RoPE(x, t)_{2i} = x_{2i} · cos(t · freqs[i]) − x_{2i+1} · sin(t · freqs[i]) RoPE(x, t)_{2i+1} = x_{2i} · sin(t · freqs[i]) + x_{2i+1} · cos(t · freqs[i])

The implementation uses PyTorch's complex number operations for efficient batched rotation:

def apply_rotary_emb(xq, xk, freqs_cis): # Reshape last dim to pairs → treat as complex numbers xq_ = torch.view_as_complex(xq.float().reshape(*xq.shape[:-1], -1, 2)) xk_ = torch.view_as_complex(xk.float().reshape(*xk.shape[:-1], -1, 2)) # Multiply by e^(iθ) = rotation in complex plane fc = freqs_cis[:seq_len].unsqueeze(1).unsqueeze(2) xq_out = torch.view_as_real(xq_ * fc).flatten(-2) xk_out = torch.view_as_real(xk_ * fc).flatten(-2) return xq_out.type_as(xq), xk_out.type_as(xk)
Why RoPE? Unlike absolute positional encodings, RoPE naturally encodes relative positions through the dot product: RoPE(q_m) · RoPE(k_n) depends only on m - n, enabling better length generalization.

4. Multi-Head Self-Attention with Causal Masking

Each attention head independently computes scaled dot-product attention with a causal (lower-triangular) mask to prevent attending to future tokens:

# Scaled dot-product attention with causal masking scores = (Q @ K.transpose(-2, -1)) / sqrt(head_dim) # [B, H, T, T] scores = scores.masked_fill(causal_mask, float('-inf')) # upper triangle → −∞ attn_weights = softmax(scores, dim=-1) # row-wise normalize output = attn_weights @ V # [B, H, T, head_dim]

Implementation details:

  • Q, K, V projections: nn.Linear(dim, dim, bias=False)
  • Output projection: nn.Linear(dim, dim, bias=False)
  • Attention dropout: 0.1 (applied after softmax)
  • Residual dropout: 0.1 (applied after output projection)
  • Causal mask: precomputed torch.triu(ones(T,T), diagonal=1).bool()

5. SwiGLU Feed-Forward Network

The SwiGLU activation replaces the standard ReLU-based FFN with a gated linear unit using the SiLU (Swish) activation:

FFN(x) = W_down( SiLU(W_gate(x)) ⊙ W_up(x) )

Where W_gate, W_up, W_down are three separate linear projections. The hidden dimension is set to 4 × dim = 512.

SwiGLU vs ReLU: SwiGLU provides smoother gradients and has been shown to improve performance in modern LLMs (PaLM, LLaMA). The gating mechanism allows the network to dynamically suppress information.

Key findings:
  • Learning rate is critical: 1e-3 far outperformed 3e-4 and 1e-4 for this task size
  • Model size vs. data: Larger models (E: 11.4M params) overfit more severely on small PTB dataset

9. Analysis & Discussion

Overfitting Pattern

The model exhibits classic overfitting behavior:

  • Train PPL drops monotonically: 484 → 25.5 (19× improvement)
  • Valid PPL reaches minimum at epoch 8 (111.94), then slowly degrades to 135.25
  • The gap between train and valid PPL grows from ~13% to ~430%, indicating memorization

Potential Improvements

  • Early stopping: Use patience-based early stopping around epoch 8-10
  • Weight tying: Share embedding and output projection weights to reduce parameters
  • Warmup schedule: Add learning rate warmup for the first ~1000 steps
  • Data augmentation: Random cropping of sequences, dropout scheduling
  • Larger dropout: Targeted dropout in embeddings and residual connections (but not too aggressive)

Reproducibility

All experiments are reproducible with fixed random seeds (seed=1234). Training scripts support full CLI configuration. Results and model checkpoints are saved automatically.
